# 测试用例文档

## 测试环境

- 操作系统：Windows 10/11
- Python 版本：3.8+（仅开发和打包环境需要）
- 终端用户环境：可不安装 Python
- 测试数据：`scheduler.json`

## 功能测试

### 1. 启动测试

| 测试项 | 操作 | 预期结果 |
|--------|------|----------|
| EXE 启动 | 双击 `DailyScheduler.exe` | 程序窗口正常打开，标题栏显示今日日期 |
| 脚本优先 EXE | 同目录存在 `start.bat` + `DailyScheduler.exe` | 双击 `start.bat` 后启动 EXE |
| 回退 Python 启动 | 无 EXE 但已安装 Python | 双击 `start.bat` 后通过 `python main.py` 启动 |
| 无 Python 且无 EXE | 双击 `start.bat` | 显示明确错误提示 |
| 首次启动 | 删除数据文件后启动 | 程序正常启动，显示空状态提示 |

### 2. 添加行程测试

| 测试项 | 操作 | 预期结果 |
|--------|------|----------|
| 正常添加 | 填写所有必填项 | 行程添加成功，列表显示新行程 |
| 空任务名称 | 任务名称留空 | 显示警告："请输入任务名称" |
| 时间格式规范化 | 输入 `9:00` 和 `10:00` | 添加成功，存储为 `09:00` 和 `10:00` |
| 时间格式错误 | 输入 `25:00` | 显示警告："Invalid time" 或等价提示 |
| 开始时间晚于结束时间 | 开始 14:00，结束 13:00 | 显示警告："结束时间必须晚于开始时间" |
| 时间相同 | 开始 14:00，结束 14:00 | 显示警告："结束时间必须晚于开始时间" |
| 优先级选择 | 选择低/中/高优先级 | 添加后优先级标签正确显示颜色 |

### 3. 时间冲突检测测试

| 测试项 | 操作 | 预期结果 |
|--------|------|----------|
| 无冲突 | 添加 09:00-10:00（已有 14:00-15:00） | 直接添加成功 |
| 完全重叠 | 添加 09:00-10:00（已有 09:00-10:00） | 提示冲突，询问是否继续 |
| 部分重叠（开始重叠） | 添加 09:30-10:30（已有 09:00-10:00） | 提示冲突，询问是否继续 |
| 部分重叠（结束重叠） | 添加 08:30-09:30（已有 09:00-10:00） | 提示冲突，询问是否继续 |
| 包含关系 | 添加 08:00-11:00（已有 09:00-10:00） | 提示冲突，询问是否继续 |
| 已完成行程不冲突 | 添加 09:00-10:00（已有已完成 09:00-10:00） | 不提示冲突，直接添加 |

### 4. 删除行程测试

| 测试项 | 操作 | 预期结果 |
|--------|------|----------|
| 正常删除 | 选中行程，点击删除 | 确认后行程被删除 |
| 取消删除 | 选中行程，点击删除后取消 | 行程保留 |
| 未选中删除 | 未选中任何行程，点击删除 | 显示提示："请先选择要删除的行程" |
| 删除最后一条 | 删除列表中唯一行程 | 删除后显示空状态提示 |
| 删除后进度 | 删除已完成/未完成行程 | 进度百分比更新 |

### 7. 数据持久化测试

| 测试项 | 操作 | 预期结果 |
